The Shipping element has a few attributes that can be used, but none are required:

checkoutpolicies – plain text or HTML encoded text, including images, to be displayed in the Shipping Method selection pane in the wizard

checkoutpolicyposition – set to top, bottom, left or right, this defines where in the wizard pane the text of the checkoutpolicies displays (top is the default)

scrolls – set to true or false, the defines whether the list of Shipping Methods should show a scoll bar or simply show all available methods (false is the default when added)

Shipping Methods has several default attributes that must always be left in place (id, label, price, u_id) and others that can be added:

countries – list of countries to display via drop down list when needsaddress fields are presented (prevents manual entry of data)

default – The default attribute can be added onto Shipping Method to pre-select that method for the user. When added it will default to true as the value, and this is the only way it needs to be added (default never needs to be set to false).

discountable – The discountable attribute can be set on a Shipping Method to prevent the price from being considered in a discount percentage calculation.

true - This Shipping Method should be discounted (this is the default condition in ROES so does not need to be listed true)

false - This Shipping Method should not be discounted

id – The id attribute is added by default on a Shipping Method and should remain there. It can be set with a value to define the accounting or internal code for the Shipping Method listed.

label – The label attribute will appear by default to be edited for the name of the Shipping Method, such as ‘UPS Ground’.

needsaddress – The needsaddress attribute can be added onto a Shipping Method to create an extra set of fields to be used for a shipping address, usually for drop shipping needs. It will also be used for calculating tax on the order.

needsaddresspolicies – Allows display of plain or HTML encoded text as instructions, etc. when the shipping address field set is presented during the completion wizard

needscounty – set to true or false, the needscounty attribute can be added onto a Shipping Method to create an extra county field in the needsaddress field set. It will also be used for calculating tax on the order, if county values are taken into consideration.

needsphone – set to true or false, the needsphone attribute can be added to a Shipping Method with needsaddress to define whether the phone field should be present or not.

price – The price attribute will appear by default to set the desired price for the Shipping Method. When used with tiertype and uses, this value can be set to a string of values to help determine the most accurate shipping price for the order.

saveaddress - The saveaddress attribute can be added onto a Shipping Method that has needsaddress set on it to retain the needsaddress data between orders.

states – The states attribute can be added onto a Shipping Method to define a list of states to be offered via needsaddress State field.

taxable – The taxable attribute can be added onto a Shipping Method to define whether or not the Shipping Method should be taxed. By default, Shipping is not taxed in ROES, so this will never be need to be set to false, only true when needed.

true - This Shipping Method should be taxed

false - This Shipping Method should not be taxed (not needed to be listed false as this is default condition in ROES)

taxrate – The taxrate attribute can be added onto a Shipping Method to set a hard tax percentage to be used for calculating tax on the order. This will still apply regardless if address data listed by the customer, and is ideally used for a ‘lab pickup’ Shipping Method where you will always look to charge tax.

tiertype – The tiertype attribute can be added onto a Shipping Method to help define the price of the method when used with uses and price. For shipping calculations generally tiertype should only be set to threshold.

u_id – The u_id attribute will appear by default on the Shipping Method and should always be left as is.

uses - The uses attribute can be added and used to help calculate the price of the Shipping Method based on the price and tiertype values. It can be set to either price or weight to match the desired calculation, such as the order price or the total of the weight attribute values on products in the order.

Label - The Label element can be added off Shipping and placed above, below, or interspersed with Shipping Methods to act as a separator or presentation of information:

border – set to R,G,B values can define a frame to encase the text for a label element

text – plain text or HTML encoded text, including images, to be displayed in the Shipping Methods selection pane of the completion wizard
